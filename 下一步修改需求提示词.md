## 需要特别注意的现有实现点

### macOS 包内资源路径

macOS 发布包中资源应该放在：

```text
MultiMaterialSlicer.app/Contents/Resources/
```

不要把 YAML、Python 脚本或后端工具错误放进：

```text
MultiMaterialSlicer.app/Contents/MacOS/
```

否则容易造成签名、查找路径和发布包行为不一致。

### 后端命令行参数

APP 调用后端时应保持：

```bash
slice_merge_tool --config <config.yaml> --output <merged_dir>
```

不要改成位置参数，除非同步修改 APP、Python 后端和文档。

### 光强到电流

UI 表示光强，底层配置/GCode 仍可能需要电流。请保持转换逻辑集中在配置生成或材料库解析处，不要在 UI 到处散落换算公式。

### 原生文件选择器

自动化工具可能不能控制 macOS 原生文件选择器。这不代表导入按钮坏了。测试自动化可以使用启动参数导入 STL/STEP，但人工验收仍要点一次“导入 STL”和“导入 STEP”。
