# 打包与会议演示指南

更新时间：2026-06-29

本文档用于会议现场演示 MultiMaterialSlicer。它包含当前已经验证过的 macOS 打包结果、Windows 打包方式、演示步骤、测试 STL/STEP 文件和注意事项。

```text
/Users/asher/Library/Mobile Documents/com~apple~CloudDocs/Documents/3D-GUI/dist/MultiMaterialSlicer-mac-x86_64.zip
```

2026-06-29 最新验证包信息：

```text
文件大小: 159 MB
SHA-256: 33512d426ae6e137f23b82d851f5893f4cc32ecdd3b9d2c58b0a425e394e644a
```

这个 zip 包内包含：

- `MultiMaterialSlicer.app`
- Qt 5.12.12 运行库
- `machine_material_presets.yaml`
- 后端命令行工具 `slice_merge_tool`
- STEP 装配体转换工具 `step_to_stl_parts`

当前 GUI 包是 x86_64 架构。Intel Mac 可以直接运行；Apple Silicon Mac 需要 Rosetta。当前这台机器生成的 STEP 转换工具是 arm64 helper，Apple Silicon 上已实测可由 x86_64 GUI 启动；如果要给 Intel Mac 分发 STEP 导入能力，建议在 Intel Mac 或 x86_64 Python 环境重新打包 `step_to_stl_parts`。

## 二、macOS 重新打包

如果会议前需要重新生成包，在仓库根目录运行：

```bash
cd "/Users/asher/Library/Mobile Documents/com~apple~CloudDocs/Documents/3D-GUI"
bash scripts/package_macos.sh
```

打包成功后会输出：

```text
dist/MultiMaterialSlicer-mac-x86_64.zip
```

推荐使用 `bash scripts/package_macos.sh`，不要直接使用 `./scripts/package_macos.sh`。项目在 iCloud Drive 路径下，直接执行脚本时可能受到文件扩展属性或执行权限影响。

## 三、macOS 打包脚本做了什么

`scripts/package_macos.sh` 会自动完成：

1. 调用 `scripts/package_backend_macos.sh`，用 PyInstaller 把 `slice_1080p.py` 打成 `backend_dist/slice_merge_tool`。
2. 如果缺少 `backend_dist/step_to_stl_parts`，调用 `scripts/package_step_macos.sh`，用 PyInstaller + CadQuery/OCP 打包 STEP 转换器。
3. 用 CMake 编译 Qt/C++ APP。
4. 把预设库 `config/machine_material_presets.yaml` 复制进 APP 的 `Contents/Resources`。
5. 把后端工具 `slice_merge_tool` 和 STEP 转换工具 `step_to_stl_parts` 复制进 APP 的 `Contents/MacOS`。
6. 调用 `macdeployqt` 打包 Qt 依赖。
7. 清理 iCloud 扩展属性、`.DS_Store` 和 AppleDouble 文件。
8. 对 Qt framework、plugin、主程序、后端工具和 STEP 转换工具进行 ad-hoc 签名。
9. 输出 zip 包。

## 四、Windows 打包

Windows 上需要准备：

- Qt 5.12.12，例如 `C:\Qt\5.12.12\msvc2017_64`
- Visual Studio C++ 编译工具
- CMake
- Python 3.10/3.11/3.12。STEP 转换器依赖 CadQuery/OCP，推荐 Python 3.12，并勾选 “Add Python to PATH”。

在 PowerShell 中运行：

```powershell
cd "项目目录"
powershell -ExecutionPolicy Bypass -File scripts\package_windows.ps1 -QtPrefix C:\Qt\5.12.12\msvc2017_64
```

脚本默认会先打包后端 `slice_merge_tool.exe`，再打包 STEP 转换器 `step_to_stl_parts.exe`，然后编译 GUI，调用 `windeployqt.exe`，检查 `qwindows.dll`、后端工具、STEP 工具和配置文件是否进入发布目录，并用 `--help` smoke test 两个命令行工具。输出：

```text
dist\MultiMaterialSlicer-win64.zip
```

会议前推荐在 Windows 展示机上运行带自检的命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\package_windows.ps1 -QtPrefix C:\Qt\5.12.12\msvc2017_64 -RunSelfTest
```

`-RunSelfTest` 会在 zip 前运行发布目录中的 `MultiMaterialSlicer.exe --selftest`。如果包内有 `examples\demo_step\step示例.step`，优先用 STEP 装配体自检；否则用两个 STL 自检。

Windows 发布目录还会包含示例文件：

```text
dist\MultiMaterialSlicer-win64\examples\demo_stl\
dist\MultiMaterialSlicer-win64\examples\demo_step\
```

如果只打包后端：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\package_backend_windows.ps1
```

如果只打包 STEP 转换器：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\package_step_windows.ps1
```

说明：当前这台机器是 macOS，已经实际打包并验证 macOS 包；Windows 包需要在 Windows 环境实机执行脚本。

## 七、会议演示步骤

### 1. 打开 APP

打开 `MultiMaterialSlicer.app`。确认界面出现：

- 左侧 3D 构建平台。
- 右侧模型列表、变换设置、材料设置。
- 材料数量默认为 3。
- 窗口可以拖拽调整大小。

### 2. 导入 STL

点击“导入 STL”，选择：

```text
demo_stl/multi_A_base_plate.stl
demo_stl/multi_B_cross_insert.stl
```

导入后左侧 3D 区域应显示模型，右侧列表会显示模型名称和材料。

### 2b. 导入 STEP 装配体

点击“导入 STEP”，选择：

```text
demo_step/step示例.step
```

导入后右侧模型树应显示：

```text
step示例.step
  实体1    材料 1
  实体2    材料 1
```

把 `实体2` 的材料数字从 1 调到 2。此时 3D 视图中第二个子实体会变成材料2颜色。再选中父节点 `step示例.step`，右侧材料下拉会禁用，X/Y/Z/旋转/缩放会启用；调整 X 或旋转时，两个子实体会作为一个装配体整体移动。

### 3. 修改模型变换

选中一个模型后，在右侧输入：

- X 平移：例如 `8.5`
- Y 平移：例如 `-4`
- Z 旋转：例如 `30`
- 等比例缩放：例如 `1.25`

输入后 3D 视图会更新模型位置、角度和大小。

### 4. 修改材料

选中模型，在材料下拉框中把模型设为材料2或材料3。模型颜色会随材料改变，模型列表也会更新为类似：

```text
multi_B_cross_insert [材料2]
```

### 5. 复制模型

选中模型，点击“复制”。列表中会出现副本，例如：

```text
multi_B_cross_insert 副本 [材料2]
```

复制件会保留原模型的变换和材料，方便现场展示多材料复制逻辑。

### 6. 切换到切片页

切换到“切片/导出”页，确认：

- 材料参数表里显示“底层光强”和“普通光强”。
- 顶部/模型相关切片设置已经放到模型设置下方。
- 后端工具路径指向 APP 内部的 `slice_merge_tool`。

### 7. 设置常规选项和高级选项

勾选：

- `挡板 (groove)`
- `刮板1 (slide_0)`

可以顺手说明：

- groove 已按会议要求翻译成“挡板”。
- slide_0、slide_1、slide_2 已翻译成“刮板1、刮板2、刮板3”。
- 这组在界面中属于“常规选项”。

继续打开“高级选项”参数表，可以演示截图里原来没有放进界面的机器/GCode 参数，例如：

- `max_height`
- `min_height`
- `z_acc_h`
- `z_speed_h`
- `clean_tank`
- `dry_tank`
- `drop_time_bottom`
- `ASS`
- `ASS_times`

这些值会写入最终 `config.yaml`。如果现场要展示“手动编辑配置再导入”，点击“导入 config.yaml”，选择手动改好的 YAML 文件，界面会回填打印设置、曝光设置、常规选项和高级选项。

### 8. 生成输出

选择输出目录，点击“导出切片并生成 GCode”。

成功后 APP 会弹窗提示：

```text
已生成切片、config.yaml 与 run.gcode。
```

输出目录中应包含：

```text
config.yaml
material_0/
material_1/
material_2/
merged/
```

`merged` 目录中应包含：

```text
config.yaml
run.gcode
若干合并后的 png
```

## 九、会议前快速自检命令

如果会议前想用命令行快速确认包没有损坏：

```bash
cd "/Users/asher/Library/Mobile Documents/com~apple~CloudDocs/Documents/3D-GUI"
rm -rf /tmp/mms_check
mkdir -p /tmp/mms_check
ditto -x -k dist/MultiMaterialSlicer-mac-x86_64.zip /tmp/mms_check
codesign --verify --deep --strict --verbose=2 /tmp/mms_check/MultiMaterialSlicer.app
open -W -n /tmp/mms_check/MultiMaterialSlicer.app --args --selftest demo_stl/multi_A_base_plate.stl demo_stl/multi_B_cross_insert.stl
```

如果最后一个命令退出码为 0，说明 APP、后端、配置库和输出链路都可用。
