# 打包与会议演示指南

本文档用于会议展示前准备、现场演示和后续交付打包。

## 1. 现在 App 在哪里

当前已经构建好的 macOS App：

```text
build-x86_64/MultiMaterialSlicer.app
```

直接打开：

```bash
open build-x86_64/MultiMaterialSlicer.app
```

带一个 STL 文件启动：

```bash
open build-x86_64/MultiMaterialSlicer.app --args demo_stl/01_basic_block.stl
```

注意：当前本机 Qt 5.12.12 是 `x86_64`，所以 Apple Silicon 上运行需要 Rosetta。

## 2. 测试 STL 文件

已经生成了会议演示用 STL：

```text
demo_stl/
  01_basic_block.stl
  02_cylinder_20mm.stl
  03_star_badge.stl
  04_three_towers.stl
  multi_A_base_plate.stl
  multi_B_cross_insert.stl
```

建议用途：

- `01_basic_block.stl`：基础导入、平移、旋转、缩放演示。
- `02_cylinder_20mm.stl`：展示圆柱类模型切片。
- `03_star_badge.stl`：展示非矩形轮廓切片。
- `04_three_towers.stl`：展示多高度模型切片层数变化。
- `multi_A_base_plate.stl` + `multi_B_cross_insert.stl`：多材料演示组合，一个设为 Material 0，一个设为 Material 1。

重新生成这些 STL：

```bash
python3 tools/generate_demo_stls.py
```

## 3. macOS 打包

### 3.1 仅本机会议演示

如果就在当前电脑上演示，不需要完整部署 Qt framework，直接用：

```bash
cmake -S . -B build-x86_64 \
  -DCMAKE_PREFIX_PATH=/Users/asher/Qt/5.12.12/clang_64 \
  -DCMAKE_OSX_ARCHITECTURES=x86_64
cmake --build build-x86_64 -j4
open build-x86_64/MultiMaterialSlicer.app
```

### 3.2 打成 zip 包

执行：

```bash
./scripts/package_macos.sh
```

输出：

```text
dist/MultiMaterialSlicer-mac-x86_64.zip
```

脚本会做：

1. CMake 配置。
2. 编译 `MultiMaterialSlicer.app`。
3. 使用 `macdeployqt` 收集 Qt 依赖。
4. 生成 zip。

说明：

- 这个包是 `x86_64`。
- Apple Silicon 机器需要 Rosetta。
- 未签名 App 在其他 Mac 第一次打开可能被 Gatekeeper 拦截，需要右键打开，或内部演示时执行：

```bash
xattr -dr com.apple.quarantine MultiMaterialSlicer.app
```

## 4. Windows 打包

Windows 是目标展示平台时，建议使用 Qt 5.12.12 + MSVC 64 位。

### 4.1 环境准备

需要安装：

- CMake
- Visual Studio 2019/2022，带 C++ 桌面开发组件
- Qt 5.12.12，例如 `C:\Qt\5.12.12\msvc2017_64`
- Python 3.10+ 或 3.12

Python 包：

```powershell
pip install -r requirements.txt
```

### 4.2 打包命令

PowerShell 执行：

```powershell
.\scripts\package_windows.ps1 -QtPrefix C:\Qt\5.12.12\msvc2017_64
```

输出：

```text
dist\MultiMaterialSlicer-win64.zip
```

脚本会做：

1. CMake Release 构建。
2. 拷贝 `MultiMaterialSlicer.exe`。
3. 拷贝 `slice_1080p.py` 和 `requirements.txt`。
4. 执行 `windeployqt.exe` 收集 Qt DLL。
5. 压缩成 zip。

### 4.3 Windows 上 Python 的处理方式

当前 App 调用 Python 合并脚本。会议演示建议采用最稳的方式：

1. 会议电脑提前安装 Python。
2. 执行 `pip install -r requirements.txt`。
3. App 切片页里确认 Python 路径，例如：

```text
C:\Users\<name>\AppData\Local\Programs\Python\Python312\python.exe
```

如果后续要做完全免安装包，可以把 Python 放到 exe 同级的 `python/python.exe`，App 会优先寻找：

```text
MultiMaterialSlicer-win64/
  MultiMaterialSlicer.exe
  python/
    python.exe
```

但 OpenCV、NumPy、PyYAML 的嵌入式 Python 打包需要额外处理，会议展示阶段不建议临时走这条路。

## 5. 会议前检查清单

建议会议前 30 分钟按这个清单跑一遍：

1. 打开 App。
2. 导入 `demo_stl/01_basic_block.stl`。
3. 确认左侧 3D 平台出现蓝色模型。

5. 切换到切片页。
6. 确认“每材料曝光参数表格”行数与材料数量一致，并已填好各材料曝光值。
7. 确认输出目录为一个可写目录。
8. 确认 Python 路径正确。
9. 点击“导出切片并生成 GCode”。
10. 导出过程中界面保持可交互（不卡死），日志实时刷新。
11. 看到完成弹窗。
12. 检查输出目录中存在：

```text
config.yaml
material_0/
material_1/
merged/run.gcode
```

## 6. 会议现场展示步骤

推荐展示节奏：5 到 8 分钟。

### 第一步：说明背景

可以这样讲：

```text
之前的流程是 Chitubox 分材料切片，然后用 Python 脚本合并多材料 PNG 和 GCode。
现在这个原型把前半段 STL 导入、材料分配、参数配置、PNG 切片、配置生成、Python 合并串成了一个 App。
```

### 第二步：打开 App

```bash
open build-x86_64/MultiMaterialSlicer.app
```

或者直接带示例模型：

```bash
open build-x86_64/MultiMaterialSlicer.app --args demo_stl/01_basic_block.stl
```

### 第三步：演示 3D 导入

操作：

1. 点击“导入 STL”。
2. 选择 `demo_stl/01_basic_block.stl`。
3. 左侧显示模型。
4. 右侧列表显示模型名称。

讲解点：

```text
左侧是轻量 OpenGL 3D 视图，右侧是模型操作栏。后续可以继续往 Chitubox 类 UI 靠近。
```

### 第四步：演示模型变换

操作：

1. 选择模型。
2. 修改 X/Y/Z。
3. 修改旋转 Z。
4. 修改等比缩放。

讲解点：

```text
每个 STL 模型都有独立的平移、旋转和等比缩放参数。
这些参数会参与后续 STL 切片，而不是只影响显示。
```

### 第五步：演示多材料

操作：

1. 导入 `multi_A_base_plate.stl`。
2. 导入 `multi_B_cross_insert.stl`。
3. 选择 `multi_A_base_plate`，材料设为 Material 0。
4. 选择 `multi_B_cross_insert`，材料设为 Material 1。

讲解点：

```text
每个模型可以设置自己的材料。导出时会按材料分别生成 PNG 文件夹。
```

### 第六步：演示导出切片和 GCode

操作：

1. 切换到“切片”页。
2. 确认材料数量、分辨率、层高、底层层数。
3. 在“每材料曝光参数表格”里给不同材料设置不同的底层/普通曝光时间与电流。
4. 确认输出目录。
5. 点击“导出切片并生成 GCode”。

讲解点：

```text
每种材料可以单独设置曝光时间和电流，这些参数会分别写进 config.yaml，并最终体现在 run.gcode 的 proj 指令里。
App 会先按材料导出切片 PNG，然后生成 config.yaml，之后命令行调用 Python 合并脚本，输出最终投影图和 run.gcode。
整个导出在后台线程执行，过程中界面不会卡死，右下角日志会实时刷新。
```

### 第七步：展示输出目录

打开输出目录：

```bash
open ~/MultiMaterialSlicerOutput
```

展示：

```text
material_0/
material_1/
config.yaml
merged/
  run.gcode
  1_0.png
  1_1.png
```

讲解点：

```text
material_0 和 material_1 是 App 从 STL 切出来的中间结果。
merged 是 Python 脚本合并后的最终打印输入。
```

## 8. 当前原型边界

会议里建议主动说明：

- 这是工作流原型，不是完整工业级切片器。
- 当前 STL 切片适合闭合网格。
- 暂未实现支撑、模型修复、岛检测、抗锯齿。
- 多材料路径规划和 GCode 生成沿用了原 Python 脚本逻辑。

已经具备、可以作为亮点展示：

- 深色科技蓝主题界面（QSS 实现），3D 视图带渐变背景、网格构建平台、增强光照，观感接近 ChituBox / Lychee。
- 每种材料独立的曝光参数（底层/普通时间与电流），一路写入 `config.yaml` 与 `run.gcode`。
- 导出全流程在后台线程执行，界面不冻结；线程退出时干净清理，不会崩溃。
- 同时兼容 ASCII 与二进制 STL，二进制 STL 允许结尾 padding。
- 切片按三角面分桶，提升大模型切片速度。
- 内置 `--selftest` 命令行自检，可脚本化验证完整导出链路（已真机实测，退出码 0）。

这会让听众知道：目前已经跑通主链路，后续重点是工程化和切片质量增强。

## 9. 自检 / 快速验证（演示前可跑一遍）

不用点界面，命令行即可验证完整导出链路是否正常：

```bash
cd "/Users/asher/Library/Mobile Documents/com~apple~CloudDocs/Documents/3D-GUI"
build-x86_64/MultiMaterialSlicer.app/Contents/MacOS/MultiMaterialSlicer \
  --selftest demo_stl/multi_A_base_plate.stl demo_stl/multi_B_cross_insert.stl
echo "退出码: $?"
```

预期：打印 `selftest: SUCCESS`，退出码 `0`，并在 `/tmp/selftest_out/` 下生成 `material_0/`、`material_1/`、`config.yaml`、`merged/run.gcode`。退出码非 0 说明链路有问题，演示前应先排查。
