# 代码生成与修改说明

更新时间：2026-06-29

本文档记录 MultiMaterialSlicer 当前版本实际做过的修改。它偏“流水账”和工程说明，方便后续向同事、老师或另一个 AI 说明现在项目已经完成到什么程度。

## 一、总体目标

最开始的工作流是：

1. 在 Chitubox 中分别为每种材料对应的模型设置位置。
2. 分别切片并导出 zip。
3. 解压成多个材料文件夹。
4. 使用 `slice_1080p.py` 把多个材料文件夹合并成一个输出文件夹和 GCode。

当前版本已经把这个流程整合进一个 Qt/C++ 桌面 APP：

1. 在 APP 中导入多个 STL，或直接导入 STEP/STP 装配体。
2. 在 3D 视图中查看模型。
3. 通过右侧参数栏调整每个模型的平移、旋转和等比例缩放。
4. 给每个模型分配材料。
5. APP 按材料导出 PNG 切片文件夹。
6. APP 生成 `config.yaml`。
7. APP 调用后端命令行工具 `slice_merge_tool` 或 `slice_merge_tool.exe`。
8. 后端生成合并后的 PNG、`run.gcode` 和最终输出目录。

## 三、界面修改

### 1. 使用 `.ui` 文件

界面定义在：

```text
ui/MainWindow.ui
```

这个文件可以用 Qt Designer 打开并继续编辑。代码侧通过 `uic` 生成界面类。

### 2. 主窗口可调节大小

之前窗口是固定大小。当前版本已经移除固定尺寸逻辑，主窗口可以拖拽调整大小。3D 区域和右侧设置栏会跟随窗口重新布局。

### 3. 左侧 3D 视图

左侧 3D 视图使用自定义控件：

```text
src/OpenGLView.h
src/OpenGLView.cpp
```

它继承自 `QOpenGLWidget`，用于显示构建平台、背景和 STL 网格。用户可以用鼠标旋转视角，用滚轮缩放视角。

最新稳定性修复中，普通模型网格会在加载后上传到 OpenGL VBO，渲染时复用 GPU buffer，不再每帧把完整顶点/法线数组从 CPU 重新传给 OpenGL。视图边缘光法线也改成视图空间计算，视角旋转时高光行为更稳定。

选择 OpenGL 的原因是：

- Qt 5.12 自带支持，不需要额外大型 3D 引擎。
- 对 STL 预览、模型变换和构建平台显示来说足够轻量。
- 迁移到 Windows、macOS、Linux 都比较直接。
- 后续如果要替换成更复杂的渲染框架，界面层和数据层已经相对分离。

### 4. 右侧模型设置

选中模型后，右侧可以输入：

- X/Y/Z 平移
- X/Y/Z 旋转
- 等比例缩放
- 材料选择

输入数值后，会更新当前选中模型，并刷新 3D 视图。

### 5. 模型复制

新增“复制”按钮。选中模型后点击复制，会复制：

- STL 网格数据
- 模型名称
- 位置
- 旋转
- 缩放
- 材料
- 显示颜色

复制后的模型名称会自动追加“副本”或编号，避免和原模型重名。

后续稳定性修复中，模型复制改为共享底层 `StlMesh` 网格数据。复制模型仍然保留独立的名称、变换、材料和颜色，但不会再为同一 STL 做一次完整顶点数据深拷贝。

### 5b. STEP 装配体导入和模型树

新增“导入 STEP”按钮。用户可以直接导入 `.step` 或 `.stp` 文件，不需要先在外部软件里把装配体拆成多个 STL。

实现方式是：

1. Qt 主程序调用 `step_to_stl_parts` 命令行转换工具。
2. 转换工具读取 STEP 文件，把每个 solid 子实体导出成一个 STL。
3. 转换工具写出 `manifest.json`，记录装配体名称、子实体名称和 STL 路径。
4. Qt 主程序读取 manifest，把每个子实体作为一个 `ModelInstance` 加入场景。
5. 主程序把这些子实体挂到同一个 STEP 父节点下面，在模型树中显示。

界面上，原来的普通列表已经改成树表：

```text
名称                  材料
step示例.step
  实体1               1
  实体2               2
```

这个树表对应 Qt Designer 中的：

```text
QTreeWidget modelList
```

第二列是材料数字控件，用户可以直接把某个子实体从材料 1 改成材料 2 或材料 3。右侧材料下拉框也会同步更新。

STEP 父节点和子节点的操作规则是：

- 选中 STEP 父节点：材料下拉框禁用，平移/旋转/缩放启用。
- 选中 STEP 子实体：材料设置启用，平移/旋转/缩放禁用。
- 修改父节点平移/旋转/缩放时，所有子实体一起移动，保持装配体相对位置。
- 修改子实体材料时，只改变该子实体的材料和显示颜色，不影响其他子实体。

为了避免破坏 STEP 装配体内部相对位置，STEP 子实体导入时不会像普通 STL 那样逐个居中。程序会先读入所有子实体 STL，然后按整个装配体的联合包围盒一起居中并落到 Z=0。

### 10. 从 config.yaml 导入参数

切片页新增“导入 config.yaml”按钮。用户可以手动编辑一个扁平 `key: value` 格式的 `config.yaml`，然后导入到界面。

导入后会回填：

- 机器名称。
- 材料数量。

这个导入功能用于“手动编辑配置再加载到 APP”，不替代 STL 导入和切片导出流程。APP 导出时仍会重新生成材料 PNG 路径和最终 `config.yaml`。

## 四、数据结构修改

主要数据结构在：

```text
src/ModelTypes.h
```

当前核心结构包括：

- `ModelInstance`：一个 STL 模型实例，包含 mesh、transform、material index、颜色等。
- `ModelInstance` 现在也可以代表 STEP 装配体的一个子实体，新增了 `isAssemblyChild`、`assemblyId`、`assemblyName` 等字段。
- `MaterialExposure`：一个材料槽对应的底层曝光、普通曝光、底层光强、普通光强、材料 preset id。
- `SliceSettings`：一次切片导出需要的全部设置，包括机器尺寸、分辨率、层厚、材料数量、材料参数、输出路径、后端工具路径、常规选项和高级机器参数等。

`MainWindow` 中新增了 STEP 装配体状态：

```text
m_assemblyTransforms
m_assemblyNames
m_assemblyOrder
```

这些数据用于记录父节点变换、显示名称和树中的显示顺序。

## 十三、自检能力

`src/MainWindow.cpp` 中加入了内置 `--selftest`。它可以让 APP 在不人工点击的情况下运行一遍关键流程：

1. 启动 APP。
2. 导入两个 STL，或导入一个 STEP 装配体并拆出两个以上子实体。
3. 设置材料。
4. 设置输出目录。
5. 勾选挡板和刮板1。
6. 导出 PNG。
7. 生成 `config.yaml`。
8. 调用后端工具。
9. 检查输出结果。
10. 返回退出码。

使用方式：

```bash
open -W -n /tmp/mms_check/MultiMaterialSlicer.app --args --selftest demo_stl/multi_A_base_plate.stl demo_stl/multi_B_cross_insert.stl
```

STEP 自检示例：

```bash
open -W -n build-x86_64/MultiMaterialSlicer.app --args --selftest demo_step/step示例.step
```

退出码为 0 表示通过。输出目录使用 Qt 的系统临时目录，例如 macOS 上是 `/private/var/.../T/mms_selftest_out`，Windows 上会落到 `%TEMP%\mms_selftest_out`。

## 十六、稳定性与性能修复

根据代码评审提出的问题，最新版本进一步做了这些修复：

- `SliceExporter` 在计算层数前检查 `maxZ`、层高和层数是否为有限值，避免 NaN 进入 `ceil` 和整数转换。
- `SliceExporter` 在切片过程中支持进度回调和取消回调。
- `StlMesh` 会拒绝 NaN/Inf 顶点，也会检查二进制 STL 声明三角数和文件长度，避免异常 STL 触发溢出或越界。
- 二进制 STL 与 ASCII STL 的判断已调整：结构合理的二进制 STL 会优先走二进制解析，不会因为 header 注释里有 `facet` 或 `vertex` 就误判。
- STL 预览加载器增加 512MB 文件大小上限，避免超大 STL 一次性 `readAll()` 触发 OOM。
- 模型树材料微调框不再捕获旧的行号，而是捕获稳定模型 ID；删除或复制模型后再改材料不会写错对象。
- `SliceWorker` 不再无限 `waitForFinished(-1)`，后端工具启动有 30 秒等待，运行有 10 分钟超时，取消时会 kill 后端进程。
- 切片页新增取消按钮和进度条。导出 PNG 阶段显示百分比，后端阶段显示不确定进度，完成后恢复按钮状态。
- STEP 转换命令行也增加超时保护。
- OpenGL 视图使用 VBO 缓存普通模型网格，并在模型移除后释放不用的 buffer。
- 底板表面、网格、边框和坐标轴几何缓存到成员变量，构建平台尺寸变化时才重建。
- 模型复制使用 `QSharedPointer<StlMesh>` 共享网格，避免大模型复制时全量深拷贝。
- 视图边缘光法线改到视图空间计算。
- 光强/电流映射插值前会过滤非有限点并排序，YAML 里的映射顺序不会影响结果。
- 机器/材料预设库加载会校验非法数字、0 尺寸、0 像素尺寸、材料上限和 current/strength 映射长度，不会再把非法值静默解析成 0。
- 材料转换参数不再只能硬编码；可以通过高级选项里的 `material_transition_enabled`、`material_transition_current`、`material_transition_time` 控制全局默认值，也可以从 `config.yaml` 导入 `m0Tom1_current` 等单向转换参数。
- 切片导出复用每个材料的一张 `QImage`，不再每层每材料重复分配大图缓冲。
- 工作流启动前会检查至少有一个可见且非空的模型。
- 开发模式 Python 路径优先使用 PATH 中的 `python3`，再回退到 Homebrew 和系统路径。
- 预设库路径支持 `--config` / `--preset-config` 命令行参数，也支持 CMake 编译期注入的源码配置路径。
- 模型和装配体内部 ID 计数器改为 64 位。

## 十七、当前限制

当前版本已经适合会议演示完整流程，但仍有一些后续可以继续加强的点：

- Windows 包脚本已经写好，但需要在真实 Windows 机器上打包和实机验证。
- macOS GUI 包当前是 x86_64，不是 universal；当前 STEP helper 在 Apple Silicon 上是 arm64，如需 Intel Mac STEP 导入能力需要重新打包 helper。
- 轻量切片器适合演示流程，后续如果追求生产精度，需要继续增强几何切片质量。
- 当前 STEP 默认转换器按 solid 拆分，已满足示例和“子实体独立材料”的演示；如果后续需要完整保留复杂 CAD 装配树的多级命名、实例复用和装配约束，可以把 `step_to_stl_parts` 替换成更完整的 XCAF/OCC 解析器，GUI manifest 接口可以继续沿用。
- APP 目前未做正式开发者签名和 notarization。
- 自动化 GUI 测试受 macOS 原生文件选择器限制，文件选择这一步仍建议人工确认。
