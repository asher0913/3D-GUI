# 代码生成与修改说明

更新时间：2026-06-25

本文档记录 MultiMaterialSlicer 当前版本实际做过的修改。它偏“流水账”和工程说明，方便后续向同事、老师或另一个 AI 说明现在项目已经完成到什么程度。

## 一、总体目标

最开始的工作流是：

1. 在 Chitubox 中分别为每种材料对应的模型设置位置。
2. 分别切片并导出 zip。
3. 解压成多个材料文件夹。
4. 使用 `slice_1080p.py` 把多个材料文件夹合并成一个输出文件夹和 GCode。

当前版本已经把这个流程整合进一个 Qt/C++ 桌面 APP：

1. 在 APP 中导入多个 STL。
2. 在 3D 视图中查看模型。
3. 通过右侧参数栏调整每个模型的平移、旋转和等比例缩放。
4. 给每个模型分配材料。
5. APP 按材料导出 PNG 切片文件夹。
6. APP 生成 `config.yaml`。
7. APP 调用后端命令行工具 `slice_merge_tool` 或 `slice_merge_tool.exe`。
8. 后端生成合并后的 PNG、`run.gcode` 和最终输出目录。

## 二、工程框架

当前工程使用：

- C++17
- CMake
- Qt 5.12.12
- Qt Widgets
- Qt Designer `.ui` 文件
- `QOpenGLWidget` 作为轻量 3D 视图
- Python + PyInstaller 打包后端命令行工具

主程序目标名：

```text
MultiMaterialSlicer
```

后端正式工具名：

```text
macOS:   slice_merge_tool
Windows: slice_merge_tool.exe
```

## 三、界面修改

### 1. 使用 `.ui` 文件

界面定义在：

```text
ui/MainWindow.ui
```

这个文件可以用 Qt Designer 打开并继续编辑。代码侧通过 `uic` 生成界面类。

### 2. 主窗口可调节大小

之前窗口是固定大小。当前版本已经移除固定尺寸逻辑，主窗口可以拖拽调整大小。3D 区域和右侧设置栏会跟随窗口重新布局。

### 3. 左侧 3D 视图

左侧 3D 视图使用自定义控件：

```text
src/OpenGLView.h
src/OpenGLView.cpp
```

它继承自 `QOpenGLWidget`，用于显示构建平台、背景和 STL 网格。用户可以用鼠标旋转视角，用滚轮缩放视角。

选择 OpenGL 的原因是：

- Qt 5.12 自带支持，不需要额外大型 3D 引擎。
- 对 STL 预览、模型变换和构建平台显示来说足够轻量。
- 迁移到 Windows、macOS、Linux 都比较直接。
- 后续如果要替换成更复杂的渲染框架，界面层和数据层已经相对分离。

### 4. 右侧模型设置

选中模型后，右侧可以输入：

- X/Y/Z 平移
- X/Y/Z 旋转
- 等比例缩放
- 材料选择

输入数值后，会更新当前选中模型，并刷新 3D 视图。

### 5. 模型复制

新增“复制”按钮。选中模型后点击复制，会复制：

- STL 网格数据
- 模型名称
- 位置
- 旋转
- 缩放
- 材料
- 显示颜色

复制后的模型名称会自动追加“副本”或编号，避免和原模型重名。

### 6. 材料数量

材料数量默认是 3。用户可以通过设置栏修改材料数量。材料槽数量改变后：

- 模型材料下拉框会更新。
- 材料参数表会更新。
- 超出范围的模型材料索引会被自动收回到有效范围内。

### 7. “电流”改为“光强”

界面上已按会议要求把“电流”改成“光强”：

- `底层光强`
- `普通光强`

注意：后端和 GCode 仍需要电流值。因此 APP 写 `config.yaml` 时，会根据机器/材料库中的强度到电流映射，把光强转换成对应电流字段。

### 8. 挡板和刮板

高级选项中已经改为中文名称：

- `挡板 (groove)`
- `刮板1 (slide_0)`
- `刮板2 (slide_1)`
- `刮板3 (slide_2)`

写入配置时仍保留后端字段名：

```yaml
groove: true
slide_0: true
slide_1: false
slide_2: false
```

## 四、数据结构修改

主要数据结构在：

```text
src/ModelTypes.h
```

当前核心结构包括：

- `ModelInstance`：一个 STL 模型实例，包含 mesh、transform、material index、颜色等。
- `MaterialExposure`：一个材料槽对应的底层曝光、普通曝光、底层光强、普通光强、材料 preset id。
- `SliceSettings`：一次切片导出需要的全部设置，包括机器尺寸、分辨率、层厚、材料数量、材料参数、输出路径、后端工具路径、高级选项等。

## 五、机器和材料库

新增预设库文件：

```text
config/machine_material_presets.yaml
```

新增加载代码：

```text
src/PresetLibrary.h
src/PresetLoader.h
src/PresetLoader.cpp
```

APP 启动时会自动查找配置库，查找顺序包括：

1. macOS APP 包内 `Contents/Resources/machine_material_presets.yaml`。
2. 可执行文件同级目录。
3. 当前工作目录下的 `config/machine_material_presets.yaml`。
4. 向上查找项目根目录的 `config/machine_material_presets.yaml`。

如果找不到，APP 会要求用户手动选择。如果仍然无法读取，则禁用导入、复制、切片等主要功能。

配置库里包含：

- 机器信息
- 构建尺寸
- 图像分辨率
- 默认材料数量
- 材料名称
- 材料默认曝光
- 光强到电流的映射

## 六、切片导出

STL 按材料导出 PNG 的逻辑在：

```text
src/SliceExporter.cpp
src/SliceExporter.h
```

它会按当前模型变换结果，把不同材料的模型分组，然后导出：

```text
material_0/
material_1/
material_2/
...
```

每个材料文件夹下是连续 PNG 图像。当前导出是项目内部轻量切片实现，适合演示完整工作流和后续工程扩展。后续如果要达到生产级切片精度，可以继续替换或加强这里的几何切片算法。

## 七、config.yaml 生成

配置文件写入逻辑在：

```text
src/ConfigWriter.cpp
src/ConfigWriter.h
```

它会生成后端需要的 `config.yaml`，包括：

- `material_num`
- `img_path_0`、`img_path_1`、`img_path_2`
- 输出图像尺寸
- 层厚
- 底层曝光、普通曝光
- 底层光强、普通光强
- 由光强换算得到的电流
- 材料切换参数
- 挡板和刮板参数

示例字段：

```yaml
material_num: 3
img_path_0: /tmp/output/material_0
img_path_1: /tmp/output/material_1
img_path_2: /tmp/output/material_2
groove: true
slide_0: true
slide_1: false
slide_2: false
```

## 八、后端调用

后端调用逻辑在：

```text
src/SliceWorker.h
src/SliceWorker.cpp
```

正式模式下，APP 不再把“运行 Python 脚本”作为用户工作流，而是调用命令行工具：

```bash
slice_merge_tool --config <config.yaml> --output <merged_dir>
```

Windows 对应：

```powershell
slice_merge_tool.exe --config <config.yaml> --output <merged_dir>
```

开发模式下，如果后端路径是 `.py` 文件，仍可以使用 Python 运行：

```bash
python3 slice_1080p.py --config <config.yaml> --output <merged_dir>
```

这样既满足当前演示中“后端是命令行或 exe”的要求，又保留了快速调试 Python 源码的能力。

## 九、Python 后端修改

原来的：

```text
slice_1080p.py
```

已经改为支持命令行参数：

```bash
python3 slice_1080p.py --config <config.yaml> --output <merged_dir>
```

主要变化：

- 支持从 `config.yaml` 读取所有输入路径和参数。
- 支持指定输出目录。
- 支持 PyInstaller 打包为单文件命令行工具。
- 增强 PNG 读取和路径处理。
- 运行完成后复制配置文件到最终输出目录，方便追踪。
- 保留原有多材料图像合并和 GCode 生成逻辑。

## 十二、打包脚本

### macOS 后端打包

```text
scripts/package_backend_macos.sh
```

功能：

- 创建 Python venv。
- 安装 `pyinstaller`、`opencv-python`、`numpy`、`pyyaml`。
- 用 PyInstaller 生成 `backend_dist/slice_merge_tool`。
- 默认生成 x86_64 后端。

### macOS APP 打包

```text
scripts/package_macos.sh
```

功能：

- 自动打包后端。
- CMake 编译 APP。
- 调用 `macdeployqt`。
- 把后端和配置库放进 APP。
- 清理 iCloud 扩展属性。
- ad-hoc 签名。
- 生成 `dist/MultiMaterialSlicer-mac-x86_64.zip`。

### Windows 后端打包

```text
scripts/package_backend_windows.ps1
```

功能：

- 创建 Python venv。
- 安装依赖。
- 用 PyInstaller 生成 `backend_dist\slice_merge_tool.exe`。

### Windows APP 打包

```text
scripts/package_windows.ps1
```

功能：

- 默认先打包后端 exe。
- 用 CMake 和 Visual Studio 工具链编译 APP。
- 调用 `windeployqt.exe`。
- 生成 `dist\MultiMaterialSlicer-win64.zip`。

## 十三、自检能力

`src/MainWindow.cpp` 中加入了内置 `--selftest`。它可以让 APP 在不人工点击的情况下运行一遍关键流程：

1. 启动 APP。
2. 导入两个 STL。
3. 设置材料。
4. 设置输出目录。
5. 勾选挡板和刮板1。
6. 导出 PNG。
7. 生成 `config.yaml`。
8. 调用后端工具。
9. 检查输出结果。
10. 返回退出码。

使用方式：

```bash
open -W -n /tmp/mms_check/MultiMaterialSlicer.app --args --selftest demo_stl/multi_A_base_plate.stl demo_stl/multi_B_cross_insert.stl
```

退出码为 0 表示通过。

## 十六、当前限制

当前版本已经适合会议演示完整流程，但仍有一些后续可以继续加强的点：

- Windows 包脚本已经写好，但需要在真实 Windows 机器上打包和实机验证。
- macOS 包当前是 x86_64，不是 universal。
- 轻量切片器适合演示流程，后续如果追求生产精度，需要继续增强几何切片质量。
- APP 目前未做正式开发者签名和 notarization。
- 自动化 GUI 测试受 macOS 原生文件选择器限制，文件选择这一步仍建议人工确认。
